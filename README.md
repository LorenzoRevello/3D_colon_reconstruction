# 3D_colon_reconstruction
Official code for "3D Reconstruction of the Colon Surface from Monocular Endoscopy for Missing Regions Quantification". This pipeline uses TSDF fusion and Poisson reconstruction to explicitly identify, map, and quantify unobserved mucosa (blind spots) from standard monocular videos, enabling objective colonoscopy quality assessment.

Endoscopic 3D Reconstruction and Coverage AnalysisThis repository provides a complete pipeline for 3D reconstruction from endoscopic sequences and advanced mesh coverage analysis. It integrates depth estimation, camera pose tracking, and a specialized module to quantify and visualize areas of the anatomy that were missed during the procedure.📂 Repository StructureThe project is organized into four main modules:depth_estimation/: Contains the Deep Learning models and scripts to predict depth maps from endoscopic frames.pose_estimation/: Includes algorithms to estimate the camera trajectory (Rotation/Translation) across the sequence.3D_reconstruction/: Features the reconstruction.py script, which uses a Scalable TSDF (Truncated Signed Distance Function) Volume to fuse depth and pose data into a high-fidelity 3D mesh or point cloud.MISSING_Region/: Features the close_mesh_poisson.py script. This module performs Poisson Surface Reconstruction to identify "holes" in the geometry, mapping and quantifying unobserved regions relative to the camera trajectory.📊 Dataset ValidationThe pipeline has been tested and validated on two major benchmark datasets for medical computer vision:C3VD: A high-fidelity clinical dataset providing ground truth for depth and camera pose.SimCol3D: A synthetic colonoscopy dataset ideal for testing reconstruction robustness in controlled environments.⚙️ Data OrganizationTo ensure the scripts (especially the TSDF integration) work correctly, organize your data as follows:Plaintext/data
  ├── sequence_name
  │   ├── Frames_metric/     # Predicted depth maps (.png) and/or RGB frames
  │   ├── SavedPosition.txt  # Camera positions (X, Y, Z)
  │   ├── SavedRotationQuaternion.txt # Camera rotations (Quaternions)
  │   ├── cam.txt            # Camera intrinsics (matrix format)
  │   └── mesh.ply           # (Output) The generated 3D reconstruction
🛠️ Usage1. 3D ReconstructionRun the TSDF integration to generate a mesh from your depth and pose data. This script includes a "Progressive Video" mode to record the reconstruction as it happens.Bashpython 3D_reconstruction/reconstruction.py
2. Missing Region AnalysisAnalyze the generated mesh to identify gaps in coverage. This script calculates geometric curvature, identifies missing regions, and generates a Bubble Graph correlating unobserved areas with trajectory difficulty.Bashpython MISSING_Region/close_mesh_poisson.py
Output Features:Mesh Repair: Fills holes using Poisson Reconstruction.Quantification: Calculates the exact area ($mm^2$) of missing regions.Visualization: Generates a colored mesh where red/orange/yellow areas indicate missing hotspots, green spheres mark centroids, and blue spheres mark the nearest trajectory points.📦 RequirementsThe project depends on several scientific and computer vision libraries. Install them using:Bashpip install -r requirements.txt
